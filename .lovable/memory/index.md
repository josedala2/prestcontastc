Sistema de Prestação de Contas do Tribunal de Contas de Angola (TCA)

## Architecture
- Frontend: React + Vite + Tailwind + shadcn/ui
- Backend: Lovable Cloud (Supabase)
- Auth: Session-based mock auth (AuthContext) with 18 roles
- Language: Portuguese (Angola)

## Key Tables
- processos: Main workflow table with 18-stage tramitação
- processo_historico: Audit trail for all stage transitions (also auto-triggered)
- processo_documentos: Documents attached to processes
- atividades: Workflow activities auto-generated by atividadeEngine
- atividade_historico: Activity state change audit trail
- profiles: User profiles (mock users seeded)
- user_roles: Role assignments using app_role enum
- submission_notifications, actas_recepcao, pareceres, submissions

## DB Functions (RPC)
- gerar_numero_processo(p_ano): Sequential process number generation
- avancar_etapa_processo(...): Atomic stage advancement with history
- estatisticas_dashboard(): Aggregated KPIs for dashboard
- estatisticas_por_perfil(p_perfil): Activity stats per role
- has_role(_user_id, _role): Security definer role check
- update_updated_at_column(): Generic trigger for updated_at
- audit_processo_changes(): Auto-audit trigger on processos

## Frontend Hooks
- src/hooks/useBackendFunctions.ts: Wrappers for all RPC functions

## Workflow
- 18 stages defined in src/types/workflow.ts
- 4 entity categories with distinct document checklists
- AtividadesTimeline component in ProcessoDetalhe

## Design
- Institutional/governmental style
- Gold accent gradients on sidebar
- serif fonts for headings
- Logo: /logo-tca.png and brasão de Angola
